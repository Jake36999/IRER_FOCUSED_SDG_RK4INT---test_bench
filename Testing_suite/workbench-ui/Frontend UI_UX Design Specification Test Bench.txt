5. Advanced Performance Optimization for Large-Scale Dashboards
Performance is a functional safety requirement. If the UI freezes during a thermal runaway, the HMI has failed its primary mission.
High-Frequency Data Handling & Batching
To prevent React from "choking" on hundreds of WebSocket messages per second, we must implement a batching strategy:
* Ref-Based Buffering: Incoming messages must be stored in a useRef rather than useState. A ref allows us to store data without triggering a render cycle on every single message, which would otherwise crash the browser thread.
* Timed Flushing: A setInterval or requestAnimationFrame loop must "flush" the buffer into the application state every 100ms. This keeps the UI responsive and smooth (60 FPS) while ensuring CPU cycles are not wasted on sub-perceptual updates.
Computational Efficiency
* Virtualization: Transaction logs and system events must use virtualization (rendering only visible rows). This ensures a table of 10,000 logs performs with the same efficiency as a table of 10.
* Web Workers: Heavy mathematical computations, such as calculating rolling averages or performing anomaly detection on the 10Hz–23kHz vibration stream, must be offloaded to Web Workers to keep the main UI thread fluid.
* Memory Management: Arrays must be capped using data.slice(-50) to prevent infinite memory growth during long-duration tests, which would lead to eventual browser instability.

const TelemetryChart = ({ data, width = 500, height = 100 }) => {

      y: height - (d.h_norm / maxVal) * height

  );
